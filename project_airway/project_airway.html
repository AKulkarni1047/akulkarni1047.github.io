Project Airway

A Brief Overview

A flight ticket booking system has become a necessity if the present world.

From personal usage to booking tickets for work related trips applications which can provide a wide variety of options to users to book flights are considered to be very important.

Our project has developed a fully functional flight booking system which can work on both mobile devices as a mobile application as well as a web application for usage on laptops or personal computers.

Modules Description

The system is partitioned into three integral modules, each tailored to orchestrate specific functionalities within the ecosystem:

  • The Mobile Application: As the front-facing component of the system, it provides the interactive mobile interface through which users apply for schemes. It is engineered for a seamless and intuitive user experience, ensuring that interactions are straightforward and secure.
  • The Web Application: As the front-facing component of the system, it provides the interactive web interface through which users apply for schemes. It is engineered for a seamless and intuitive user experience, ensuring that interactions are straightforward and secure.
  • The Backend Module: The Node.js and MongoDB-powered backend of our flight ticket booking app enables users to create accounts, log in securely, and seamlessly book or cancel flights. The system also incorporates a reward points system, fostering user engagement and loyalty.

Technology Highlights

The system is built using various tools and platforms which are used to create a seamless experience for the user. The technologies used are:

  • Jira: Tool used to track bugs, and organize features and improvments in the application.
  • Selenium: Cloud-based testing platform which helped test the react frontend in rapid intervals.
  • Postman: Tool used to perform automated tests on the APIs developed for backend services.
  • Git: Version control tool used to streamlie process of development and testing.
  • Heroku: A PaaS that was used to deploy and manage the application.
  • Docker: Cloud based service used to containerize the backend server.
  • CircleCI: Cloud based service used to build a CICD platform to ensure updates and changes were deployed to Heroku automatically.
